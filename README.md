## 개발자가 되고 싶은

React 학습 로드맵 체크표 ✅

하늠의 리액트 성장 로드맵 🚀
README에 넣고 하나씩 체크하면서 실력 업그레이드하자 💪

🥇 1단계: 필수 기초 (React의 근육)

리액트의 작동 원리와 기초 훅 개념을 완벽히 익히는 단계

 props — 부모 → 자식 데이터 전달

 조건부 렌더링 — 상태에 따라 다른 UI 표시 (&&, ?:, return null)

 배열 렌더링 & key — .map()으로 리스트 렌더링

 useState — 상태(state) 관리

 useRef — DOM 접근 / 렌더 영향 없는 값 저장

 useEffect — 사이드 이펙트 처리 (API 호출, 이벤트 등록 등)

 useMemo / useCallback — 렌더링 최적화

 React.memo — props 변화 없을 때 리렌더 방지

 useReducer — 복잡한 상태 전이 관리

 커스텀 훅 (Custom Hook) — 훅 로직 재사용

 Context API — 전역 상태 공유, props drilling 해소

 Immer — 불변성 쉽게 유지 (produce())

📘 목표:
토이 프로젝트를 혼자 완성할 수 있을 정도로 리액트의 기본기를 다진다.

🥈 2단계: 실무 확장 (React의 신경계)

실제 서비스 개발에서 자주 사용되는 패턴과 라이브러리를 익히는 단계

 React Router (v6) — SPA 페이지 전환 / 중첩 라우팅

 React Query (TanStack) — 서버 상태 관리, 캐싱, 자동 리패치

 Zustand / Redux Toolkit — 전역 상태 중앙 집중 관리

 React Hook Form — 폼 입력값 관리 및 유효성 검사

 React.lazy / Suspense — 코드 스플리팅, 지연 로딩

 Error Boundary — UI 에러 안전 처리

 Controlled / Uncontrolled Components — 폼 제어 패턴 이해

 Portal — 모달, 드롭다운 등 DOM 외부 렌더링

 Fragment / key 관리 — 불필요한 DOM 감싸기 제거 및 성능 최적화

📘 목표:
로그인, 상품 리스트, 마이페이지, 리뷰 등
중형 규모 서비스 구현이 가능한 수준까지 확장!

🥉 3단계: 심화 & 퍼포먼스 (React의 엔진룸)

리액트의 내부 렌더링 원리와 최적화 기법을 다루는 고급 단계

 Virtual DOM / Reconciliation — React 렌더링 원리 이해

 useTransition / useDeferredValue — React 18 동시 렌더링, 우선순위 제어

 Suspense for Data Fetching — 비동기 UI 제어

 코드 스플리팅 최적화 — dynamic import로 초기 로딩 속도 개선

 React Fiber 구조 — 렌더링 스케줄링 엔진 이해

 Jest / React Testing Library — 단위 테스트, 컴포넌트 테스트

 TypeScript Props 제너릭 — 컴포넌트 타입 안정성 확보

 Vite / Webpack / Babel — 번들링 및 빌드 구조 이해

 ESLint / Prettier — 협업 시 코드 일관성 유지

대규모 프로젝트의 성능 최적화 + 유지보수 + 배포 구조까지 다루는 수준 도달! 🚀
